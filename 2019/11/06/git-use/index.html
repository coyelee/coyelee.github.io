Git常用命令

发表于 2019-11-06 分类于 Tool

命令列表

初始化

  • 在当前目录新建一个Git代码库

  • 下载一个项目和它的整个代码历史 [Git only]

配置

  • 列举所有配置

  • 为命令配置别名

  • 设置提交代码时的用户信息

增删文件

  • 添加当前目录的所有文件到暂存区

  • 添加指定文件到暂存区

  • 添加指定目录到暂存区，包括其子目录

  • 删除工作区文件，并且将这次删除放入暂存区

  • 停止追踪指定文件，但该文件会保留在工作区

  • 改名文件，并且将这个改名放入暂存区

分支

  • 列出所有本地分支

  • 列出所有本地分支和远程分支

  • 新建一个分支，但依然停留在当前分支

  • 新建一个分支，并切换到该分支

  • 切换到指定分支，并更新工作区

  • 合并指定分支到当前分支

  • 选择一个 commit，合并进当前分支

  • 删除本地分支，-D 参数强制删除分支

  • 删除远程分支

  • 在本地创建和远程分支对应的分支

  • 建立本地分支和远程分支的关联

提交

  • 提交暂存区到仓库区

  • 提交工作区与暂存区的变化直接到仓库区

  • 提交时显示所有 diff 信息

  • 提交暂存区修改到仓库区，合并到上次修改，并修改上次的提交信息

  • 上传本地指定分支到远程仓库

拉取

  • 下载远程仓库的所有变动 (Git only)

  • 显示所有远程仓库 (Git only)

  • 显示某个远程仓库的信息 (Git only)

  • 增加一个新的远程仓库，并命名 (Git only)

  • 取回远程仓库的变化，并与本地分支合并，(Git only), 若使用 Git-SVN，请查看第三节

  • 取回远程仓库的变化，并与本地分支变基合并，(Git only), 若使用 Git-SVN，请查看第三节

  • 拉取远程代码后执行rebase，整理提交线

  • 拉取后如果有冲突，先本地解决冲突后，重新

撤销

  • 恢复暂存区的指定文件到工作区

  • 恢复暂存区当前目录的所有文件到工作区

  • 恢复工作区到指定 commit

  • 重置暂存区的指定文件，与上一次 commit 保持一致，但工作区不变

  • 重置暂存区与工作区，与上一次 commit 保持一致

  • 重置当前分支的指针为指定 commit，同时重置暂存区，但工作区不变

  • 重置当前分支的HEAD为指定 commit，同时重置暂存区和工作区，与指定 commit 一致

  • 新建一个 commit，用于撤销指定 commit

  • 将未提交的变化放在储藏区

  • 将储藏区的内容恢复到当前工作区（stash被删除）

  • 将储藏区的内容恢复到当前工作区（stash未被删除）

  • 将stash后的内容上删除

  • 恢复某个stash

查询

  • 查看工作区文件修改状态

  • 查看工作区文件修改具体内容

  • 查看暂存区文件修改内容

  • 查看版本库修改记录

  • 查带表日志

  • 可以查看分支合并吐

  • 查看某人提交记录

  • 查看某个文件的历史具体修改内容

  • 查看某次提交具体修改内容

添加

  • 关联本地于远程仓库

  • 推送本地仓库到远程仓库

  • 分支合并带分支合并记录

  • 查看某个tag信息

  • 查看所有tag

  • 删除某个tag

  • 从本地把创建的tag推到远程

  • 一次性推送全部尚未推送到远程的本地标签
